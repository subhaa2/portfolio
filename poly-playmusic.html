PlayMusic!

A Year 2 Semester 1 school project that turned front-end and database concepts into a personalised audio web application with user accounts, playlists, and library management.

Landing page for PlayMusic! introducing the audio web application and its main navigation.

Course project note: this project was completed in Year 2 Semester 1 over about four weeks for the Open Source Web Solutions Development and Web Design and Development modules.

PlayMusic! was a school-related audio web application project built during Year 2 Semester 1. The goal was to create a personalised media application using the languages we had learned across the modules, including HTML, CSS, PHP, JavaScript, jQuery, and MySQL.

I created an audio web application where users could create an account, log in to view their playlist, add songs from the library into that playlist, remove songs they no longer wanted, and update their email information if it changed. It took about four weeks to complete and was one of the more enjoyable projects to build because it combined interface work with database-backed functionality.

Key features

  • User account creation and login for personalised access.
  • Playlist management so users could review saved songs in one place.
  • Library browsing with the ability to add featured songs to the personal playlist.
  • Delete actions for removing songs from the playlist when needed.

Account workflow

  • Registered users could sign in and reach their own playlist view.
  • The application supported updating email details in the account section.
  • Different screens worked together as one personalised user journey rather than isolated pages.

Development approach

  • Combined front-end structure and styling with server-side PHP logic.
  • Used JavaScript and jQuery to support interaction across the application flow.
  • Stored and retrieved user and playlist data with a MySQL-backed setup.
