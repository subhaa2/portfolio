J.P. Morgan Internship

Database Monitoring

An internal monitoring website that gave staff visibility into database health, growth trends, and operational risk signals.

Confidentiality note: the original dashboard layout and database names are simplified for this portfolio version.

The Database Monitoring project provided internal staff with valuable information about the health and growth of important databases. It turned scattered operational metrics into a structured website that teams could check regularly to identify issues before they became larger problems.

Using pre-built components, APIs, Python functions, and SQL queries, the site surfaced insights such as VM-to-RAM ratios, directory sizes, growth rates, and process activities. The overall goal was proactive monitoring that supported timely interventions and stronger operational integrity.

Python SQL APIs Monitoring
Period

Audience
Internal operations staff

Core Outcome
Proactive intervention

What it monitored

  • Database health indicators and growth patterns across important internal systems.
  • VM-to-RAM ratios to support infrastructure awareness and capacity review.
  • Directory sizes, growth metrics, and process activity for more complete operational visibility.

Interface highlights

  • Database selection controls and date-based filtering for focused analysis.
  • Tables for machine, host, and metric comparisons across monitored systems.
  • Sections dedicated to directory size tracking, total growth, average growth rate, and running processes.

Technical contribution

The site brought together reusable internal components, API data, Python functions, and SQL queries into a single monitoring surface. The engineering challenge was to make multiple operational signals readable in one workflow so that teams could act quickly when something drifted out of range.
